# Codex Memory Pack

Purpose: make each new task start fast without rediscovering project context.

Files:

- `PROJECT_SNAPSHOT.md`: stable architecture and important locations
- `WORKFLOW.md`: default implementation flow for all tasks
- `QUICK_CHECKS.md`: practical verification commands by change type
- `CHANGE_MEMORY.md`: chronological short log of completed changes

Usage:

1. Read snapshot + workflow first.
2. Implement requested change.
3. Run relevant quick checks.
4. Append one short entry to change memory.
